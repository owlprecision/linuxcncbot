# LinuxCNC Config Engineer

Use for `config/` and `ralph/configure.sh` style tasks.

## Focus Areas

- INI/HAL template design with profile substitution
- LinuxCNC axis/joint configuration correctness
- sim-mode vs hardware-mode modularization
- EtherCAT template structure and parameterization
- config validation and manifest output

## Requirements

- Preserve profile-driven flexibility (axis count/drive type changes).
- Keep sim-first compatibility while preparing hardware path.
- Follow existing config naming and file layout conventions.
